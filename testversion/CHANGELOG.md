# CHANGELOG - 雀魂金玉四麻风格分析助手

## 项目概述

一个基于 Tampermonkey 的雀魂对手风格实时分析工具，通过牌谱屋公开 API 获取玩家历史数据，实时分析并显示对手的打牌风格。

**开发时间**：2026-03-11
**开发工具**：Claude Code (Sonnet 4.6)
**开发方式**：对话式编程，AI 辅助全程开发和调试
**代码量**：468 行纯 JavaScript (ES5)

#### 3. 风格分析算法

```javascript
// 进攻意愿 = 立直率 + 副露率
var 进攻意愿 = 立直率 + 副露率;
var 基准进攻意愿 = baseline.立直率 + baseline.副露率;

// 进攻效率 = (和牌率 / 进攻意愿) × 100
var 进攻效率 = (和牌率 / 进攻意愿) * 100;

// 偏差计算
var 进攻意愿偏差 = 进攻意愿 - 基准进攻意愿;
var 放铳率偏差 = 放铳率 - baseline.放铳率;

// 分类
if (进攻意愿偏差 > 1.5) 意愿类型 = '高';
else if (进攻意愿偏差 < -1.5) 意愿类型 = '低';
else 意愿类型 = '中';

if (放铳率偏差 < -2.0) 防守类型 = '铁壁';
else if (放铳率偏差 > 2.0) 防守类型 = '漏勺';
else 防守类型 = '正常';

// 称号映射
var 主称号 = 称号映射[意愿类型 + 防守类型];
```

### 关键突破点

#### 1. unsafeWindow 的发现

**问题**：脚本加载但 `window.view` 是 `undefined`

**原因**：Tampermonkey 默认在沙箱环境中运行，无法访问页面的真实 window 对象

**解决**：

```javascript
// @grant unsafeWindow
var gameWindow = (typeof unsafeWindow !== 'undefined') ? unsafeWindow : window;
```

#### 2. 合规性方案

**问题**：GitHub 上的项目都因为反爬被封

**方案**：

- 不解析 WebSocket → 读取游戏暴露的对象
- 不破解协议 → 使用第三方公开 API
- 不修改游戏 → 只读取，不写入

#### 3. 双轴分析模型

**灵感**：用户提出的"杠铃策略"理论

**实现**：

- X轴：进攻意愿（高/中/低）
- Y轴：防守能力（铁壁/正常/漏勺）
- 结果：3×3 = 9种风格称号
