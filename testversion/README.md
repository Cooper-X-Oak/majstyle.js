# 雀魂金玉之间四麻风格分析助手

一个基于 Tampermonkey 的雀魂对手风格实时分析工具，专注于金之间和玉之间的四人麻将对局。

## ✨ 功能特点

- 🎯 **实时风格分析**：自动识别对局玩家，分析其历史数据
- 📊 **数据可视化**：半透明悬浮窗显示关键数据，不遮挡游戏画面
- 🏷️ **风格标签系统**：9种主称号 + 11种特征标签，快速了解对手
- 📈 **偏差对比**：所有数据对比段位平均值，颜色标注偏差程度
- ⚡ **快速响应**：进入对局后2-3秒内完成分析

## 🎮 支持范围

- **对局类型**：仅支持四人麻将（四麻）
- **段位范围**：金之间、玉之间（士3-魂1）
- **数据来源**：牌谱屋（Amae Koromo）公开 API

## 📦 安装方法

### 1. 安装 Tampermonkey

- Chrome/Edge: [Tampermonkey 官网](https://www.tampermonkey.net/)
- Firefox: [Firefox 插件商店](https://addons.mozilla.org/firefox/addon/tampermonkey/)

### 2. 安装脚本

1. 点击 Tampermonkey 图标 → 管理面板
2. 点击"+"号创建新脚本
3. 复制 `雀魂对手风格分析助手-正式版.user.js` 的全部内容
4. 粘贴到编辑器，保存（Ctrl+S）

### 3. 使用

1. 打开雀魂游戏（https://game.maj-soul.com 或 https://game.maj-soul.net）
2. 进入金之间或玉之间的四麻对局
3. 等待2-3秒，自动显示分析结果

## 🔒 合规性说明

### 为什么 GitHub 上的雀魂项目都挂了？

大部分开源项目试图：

- 直接解析雀魂的 WebSocket 通信
- 破解 protobuf 协议
- 注入代码修改游戏行为

这些都会触发雀魂的反作弊检测，导致封号或项目失效。

### 本项目的合规方案

✅ **完全合规，不触碰任何反爬机制**

1. **获取玩家 ID**：
   
   - 通过浏览器开发者工具观察游戏对象
   - 读取 `window.view.DesktopMgr.Inst.player_datas`
   - 这是游戏本身暴露的公开数据，不涉及破解

2. **数据来源**：
   
   - 调用牌谱屋（Amae Koromo）的公开 API
   - 这是第三方合法的数据统计网站
   - API 地址：`https://5-data.amae-koromo.com/api/v2/pl4/`

3. **完全绕过雀魂**：
   
   - ❌ 不直接与雀魂服务器交互
   - ❌ 不解析 WebSocket
   - ❌ 不破解 protobuf
   - ❌ 不修改游戏数据
   - ❌ 不拦截游戏通信

### 核心逻辑

```javascript
// 1. 检测游戏对象
gameWindow.view.DesktopMgr.Inst.player_datas

// 2. 获取玩家数据
GM_xmlhttpRequest({
  url: 'https://5-data.amae-koromo.com/api/v2/pl4/player_stats/...'
})

// 3. 分析风格
进攻意愿 = 立直率 + 副露率
进攻效率 = (和牌率 / 进攻意愿) × 100

// 4. 显示 UI
document.body.appendChild(container)
```

## 📝 使用限制

### 数据要求

- 玩家需要在金之间/玉之间有至少 **50 局**对局记录
- 数据来自牌谱屋，部分新号可能查询不到

### 不支持的情况

- ❌ 三人麻将（三麻）
- ❌ 铜之间、银之间等低段位
- ❌ 段位战以外的模式（友人战、比赛场等可能无数据）
- ❌ 没有在牌谱屋收录的玩家

### 显示说明

- 有数据的玩家：显示完整分析
- 无数据的玩家：显示"无牌谱数据"提示
- 电脑玩家：显示"[电脑]"标记

## 🐛 常见问题

**Q: 为什么有的玩家显示"无牌谱数据"？**
A: 可能是新号、没打过金之间、或对局数不足 50 局。

**Q: 数据多久更新一次？**
A: 牌谱屋的数据通常有几小时到一天的延迟，不是实时的。

**Q: 会被封号吗？**
A: 不会。脚本完全不触碰雀魂的反作弊机制，只读取公开数据。

**Q: 为什么加载有点慢？**
A: 需要为每个玩家请求两次 API（基础数据 + 扩展数据），网络延迟正常。

**Q: 可以用于三麻吗？**
A: 不可以。所有基准数据都是基于四麻金玉之间，三麻的数据分布完全不同。
