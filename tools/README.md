# 开发工具

本目录包含用于开发和维护项目的工具。

## 工具列表

### 1. Git Push 灵活网络处理脚本

**文件**: `scripts/git-push-flexible.sh`

**用途**: 在网络不稳定或代理问题时，自动尝试多种方式推送到 GitHub

**使用方法**:

```bash
# 方式 1: 使用 npm script（推荐）
npm run push

# 方式 2: 直接运行脚本
bash tools/scripts/git-push-flexible.sh

# 方式 3: 传递额外参数
npm run push -- --force
```

**工作原理**:

1. 尝试使用当前设置推送
2. 如果失败，尝试使用代理（http://127.0.0.1:7897）
3. 如果代理失败，尝试直连
4. 推送成功后恢复原始代理设置

**特性**:

- 自动检测网络连接
- 智能切换代理/直连
- 保存并恢复原始代理设置
- 彩色输出，清晰显示每步状态

### 2. 数据探索工具

**文件**: `api-explorer.html`, `phase1-verification.js`

**用途**: 探索雀魂游戏内部对象结构和牌谱屋 API 数据

## ⚠️ 重要提示

由于浏览器的 CORS（跨域资源共享）限制，`api-explorer.html` 工具无法直接从本地文件访问牌谱屋 API。

**推荐使用游戏内探索模式**（见下方），它使用 Tampermonkey 的 `GM_xmlhttpRequest` 可以绕过 CORS 限制。

## 🎮 游戏内探索模式（推荐）

**用途**: 探索雀魂游戏内部对象结构和牌谱屋 API 数据

**使用方法**:

1. 在雀魂游戏中安装并启用用户脚本
2. 进入对局界面（必须在对局中，不能在大厅）
3. 按下 `Ctrl+Alt+E` 触发数据探索模式
4. 打开浏览器控制台（F12）查看输出
5. 将结果记录到相应的文档中

**输出内容**:

- `view` 对象的完整属性列表
- `view.DesktopMgr.Inst` 对象的属性
- `player_datas` 数组的结构
- `GameMgr.Inst` 对象的属性
- 可能的事件系统
- **当前玩家的完整 API 数据**（包括所有字段）

**注意事项**:

- 必须在对局中才能触发，大厅中无效
- 控制台输出较多，建议清空控制台后再触发
- 如果没有输出，检查脚本是否正确安装和启用

## api-explorer.html（备用方案）

**用途**: 探索牌谱屋 API 的完整数据结构

**限制**: 由于 CORS 限制，此工具只能在以下情况使用：
1. 通过本地 Web 服务器运行（如 `python -m http.server`）
2. 使用浏览器扩展禁用 CORS（不推荐）
3. 仅作为 UI 参考，实际数据通过游戏内探索模式获取

**替代方案**: 直接使用游戏内探索模式（`Ctrl+Alt+E`），它会自动获取并显示 API 数据

## 常见问题

**Q: Git 推送失败怎么办？**

A: 使用 `npm run push` 自动尝试多种网络方式

**Q: API 请求失败怎么办？**

A: 检查网络连接，确认账号 ID 正确，尝试使用其他账号 ID

**Q: 游戏对象探索没有输出？**

A: 确保在对局中（不是大厅），检查控制台是否有错误信息

**Q: 如何找到更多 API 端点？**

A: 使用浏览器开发者工具的网络面板，观察游戏和牌谱屋的网络请求

**Q: 数据探索完成后做什么？**

A: 参考 `docs/development/IMPLEMENTATION_ROADMAP.md` 继续实施后续功能
