## 🎯 项目目标

开发一个 Tampermonkey 用户脚本，能够：

1. 实时分析雀魂对局中的对手打牌风格
2. 基于牌谱屋历史数据进行统计分析
3. 在游戏界面上直观显示分析结果
4. 完全合规，不触碰雀魂的反作弊机制

## 📊 数据分析模型

### 风格分类系统

#### 双轴模型

```
进攻维度：
- 意愿 = 立直率 + 副露率
- 效率 = (和牌率 / 意愿) × 100

防守维度：
- 放铳率偏差
```

#### 9种主称号

| 进攻意愿 \ 防守 | 铁壁   | 正常  | 漏勺   |
| --------- | ---- | --- | ---- |
| **高**     | 钢铁战士 | 狂战士 | 自爆兵  |
| **中**     | 忍者   | 上班族 | 赌徒   |
| **低**     | 乌龟   | 摆烂人 | 送分童子 |

#### 11种特征标签

- 精准狙击（效率高）
- 速攻流（副露率>38%）
- 立直狂（立直率>25%）
- 大牌猎人（打点高）
- 对攻狂魔（追立率>25%）
- 慢热型（意愿低）
- 铁壁（放铳率低）
- 漏勺（放铳率高）
- 先制王（先制率>85%）
- 好型立直（好型率>60%）
- 赌博立直（好型率<40%且立直率>20%）

### 段位基准数据

基于牌谱屋统计的金之间、玉之间大数据（士3-魂1）

#### 3. 风格分析

```javascript
function analyzeStyle(stats, baseline) {
    // 计算进攻意愿和效率
    var 进攻意愿 = 立直率 + 副露率;
    var 进攻效率 = (和牌率 / 进攻意愿) * 100;

    // 计算偏差
    var 进攻意愿偏差 = 进攻意愿 - (baseline.立直率 + baseline.副露率);
    var 放铳率偏差 = 放铳率 - baseline.放铳率;

    // 分类
    var 意愿类型 = 进攻意愿偏差 > 1.5 ? '高' : 进攻意愿偏差 < -1.5 ? '低' : '中';
    var 防守类型 = 放铳率偏差 < -2.0 ? '铁壁' : 放铳率偏差 > 2.0 ? '漏勺' : '正常';

    // 映射称号
    var 主称号 = 称号映射[意愿类型 + 防守类型];

    return { 主称号, 标签, 数据, 偏差 };
}
```

### 踩过的坑

#### 1. unsafeWindow 问题

**问题**：脚本加载但 `window.view` 是 `undefined`
**原因**：Tampermonkey 默认在沙箱环境运行
**解决**：使用 `@grant unsafeWindow` 访问真实 window
**教训**：理解运行环境的限制

#### 2. ES6 兼容性

**问题**：语法错误（const/let/箭头函数）
**原因**：Tampermonkey 默认 ES5 环境
**解决**：全部改为 ES5 语法
**教训**：确认目标环境的语言版本

#### 3. 异步处理

**问题**：不能使用 async/await
**原因**：ES5 不支持
**解决**：使用 Promise 链式调用
**教训**：选择合适的异步方案

#### 4. iframe 上下文

**问题**：脚本在主窗口，游戏在 iframe
**原因**：页面结构复杂
**解决**：使用 unsafeWindow 访问
**教训**：理解页面结构

#### 5. 数据缺失

**问题**：部分玩家无牌谱数据
**原因**：新号或对局数不足
**解决**：添加错误处理和提示
**教训**：考虑边界情况

## 💡 关键技术突破

### 1. unsafeWindow 的发现

这是整个项目最关键的突破。最初脚本加载但不执行，通过调试发现 Tampermonkey 的沙箱机制导致无法访问页面的真实 window 对象。使用 `unsafeWindow` 解决了这个问题。

### 2. 合规性方案

GitHub 上的项目都因为反爬被封，我们通过：

- 读取游戏暴露的对象（不解析 WebSocket）
- 使用第三方公开 API（不直接请求雀魂）
- 只读取不修改（不触碰游戏逻辑）

完全绕过了反爬机制。

### 3. 双轴分析模型

用户提出的"杠铃策略"理论启发了双轴分析模型：

- X轴：进攻意愿（高/中/低）
- Y轴：防守能力（铁壁/正常/漏勺）
- 结果：3×3 = 9种风格称号

这个模型简单直观，易于理解和实现。
