# 测试脚本说明

本目录包含项目开发过程中的所有测试脚本，用于验证各个功能模块的可行性。

## 📋 测试脚本清单

### 01-测试获取玩家ID.user.js
**目的**：验证能否从游戏中获取玩家ID

**测试内容**：
- 访问 `window.view.DesktopMgr.Inst.player_datas`
- 获取 `window.GameMgr.Inst.account_id`
- 识别自己和其他玩家

**预期结果**：
```
✅ 成功获取玩家数据！
我的ID: 14766635
玩家数量: 4
座位0: { nickname: '...', account_id: ..., isSelf: true }
...
```

**测试结论**：✅ 通过 - 可以成功获取游戏中的玩家ID

### 02-测试牌谱屋API基础.user.js
**目的**：验证牌谱屋 player_stats API 是否可用

**测试内容**：
- 调用 `player_stats` 端点
- 测试 GM_xmlhttpRequest 的 CORS 绕过
- 验证返回数据格式

**API 端点**：
```
https://5-data.amae-koromo.com/api/v2/pl4/player_stats/{accountId}/{startTime}/{endTime}?mode=12.9&tag=492541
```

**返回数据**：
```json
{
  "account_id": 14766635,
  "nickname": "表意的符号",
  "level": {
    "id": 10403,
    "score": 1693
  }
}
```

**测试结论**：✅ 通过 - API 可用，CORS 绕过成功

---

## 🎯 测试总结

### 测试顺序
1. **01-获取玩家ID** → 验证数据源可访问
2. **02-基础API** → 验证外部API可用
3. **03-扩展API** → 确认数据字段完整
4. **04-完整流程** → 验证整体方案可行

### 关键发现

#### 1. unsafeWindow 的必要性
最初测试时发现 `window.view` 是 `undefined`，通过添加 `@grant unsafeWindow` 解决。

#### 2. API 数据丰富度
扩展统计API提供了50+个数据字段，远超预期，为风格分析提供了充足的数据支持。

#### 3. CORS 绕过方案
`GM_xmlhttpRequest` 成功绕过了CORS限制，可以直接调用牌谱屋API。

#### 4. 异步处理方案
Promise 链式调用在 ES5 环境下工作正常，可以作为异步处理方案。

### 测试环境
- **浏览器**：Chrome/Edge
- **Tampermonkey**：最新版
- **测试对局**：人机对局 + 真人对局
- **测试时间**：2026-03-10

### 测试结果
| 测试项 | 结果 | 说明 |
|---|---|---|
| 获取玩家ID | ✅ 通过 | 可以访问游戏对象 |
| 基础API | ✅ 通过 | player_stats 可用 |
| 扩展API | ✅ 通过 | player_extended_stats 可用，50+字段 |
| 完整流程 | ✅ 通过 | 整体方案可行 |

## 📝 使用说明

### 如何运行测试
1. 在 Tampermonkey 中创建新脚本
2. 复制对应测试脚本的内容
3. 保存并启用脚本
4. 打开雀魂游戏
5. 查看浏览器控制台输出

### 测试顺序建议
建议按照 01 → 02 → 03 → 04 的顺序运行测试，每个测试通过后再进行下一个。

### 注意事项
- 测试脚本会在控制台输出详细日志
- 测试03和04需要网络连接到牌谱屋
- 测试01和04需要进入对局才能看到结果
- 所有测试脚本都是一次性的，测试成功后会自动停止
