# CHANGELOG

## [2.1.0-beta.1] - 2026-03-15 (测试版 - Phase 1 完成)

### Phase 1: 增强统计分析系统

⚠️ **这是测试版本，用于调试和验证功能，不是正式发布版本**

#### 新增功能

**玩家原型检测系统**
- 🎯 **6种核心原型**: 立直专家、副露专家、默听猎手、速攻型、价值型、防守型
- 🔍 **智能匹配算法**: 基于多维度条件自动识别玩家打法风格
- 📊 **匹配度评分**: 显示原型匹配置信度
- 🎨 **原型图标**: 每种原型配有专属 emoji 图标，UI 显示更直观

**10维度危险度评估系统**
- ⚡ **增强危险度计算**: 从 4 维度扩展到 10 维度
  1. 净打点效率 (30% 权重) - 综合实力
  2. 默听率 (15% 权重) - 隐蔽性
  3. 立直收支 (20% 权重) - 立直质量
  4. 平均铳点 (5% 权重) - 防守弱点
  5. 被炸率 (10% 权重) - 大牌防守
  6. 和了巡数 (10% 权重) - 速度威胁
  7. 立直后和牌率 (5% 权重) - 立直效率
  8. 副露后和牌率 (3% 权重) - 副露效率
  9. 和牌率 (2% 权重) - 基础进攻能力
  10. 样本量置信度 - 动态权重调整
- 📈 **置信度系统**: 根据对局数量调整评分可靠性
  - 400+ 局: 100% 置信度
  - 200-399 局: 95% 置信度
  - 100-199 局: 90% 置信度
  - 50-99 局: 80% 置信度
- 🎯 **归一化评分**: 所有维度统一归一化到 0-10 分，确保公平性

**原型驱动的策略建议**
- 🧠 **上下文感知建议**: 根据检测到的玩家原型生成针对性建议
- 📋 **原型专属建议库**: 每种原型配有 3 条专属战术建议
- 🔄 **智能去重**: 避免重复建议，优先显示原型特定建议
- 📊 **建议优先级**: 原型建议 > 综合实力建议 > 细节建议

**API 字段完整映射**
- 📚 **51 个字段定义**: 在 constants.js 中建立完整的 API 字段映射表
- 🏷️ **中文字段名**: 所有字段配有清晰的中文标识
- 📦 **分类组织**: 按基础统计、效率指标、条件统计、时序指标、防守指标分类

#### 改进

**UI 增强**
- 🎨 **原型图标显示**: 在玩家信息卡标题中显示原型图标和名称
- 📊 **置信度显示**: 危险度评分旁显示置信度百分比
- 🎯 **匹配度显示**: 控制台输出原型匹配度评分

**控制台输出增强**
- 📈 **原型信息**: 显示检测到的玩家原型和匹配度
- 🔍 **置信度信息**: 显示危险度评分的置信度
- 📊 **维度得分**: 可查看各维度的详细得分（调试用）

**代码架构优化**
- 🏗️ **新增模块**: `archetype-detector.js` - 专门处理原型检测
- 📦 **配置扩展**: `constants.js` 新增原型定义和危险度权重配置
- 🔧 **函数重构**: `calculateDangerLevel()` 完全重写，支持 10 维度评估
- 🎯 **策略生成**: `generateStrategies()` 重构，支持原型驱动建议

#### 技术改进

- 重构 `advice-generator.js`（239 行 → ~350 行）
- 新增 `archetype-detector.js`（~130 行）
- 扩展 `constants.js`（44 行 → ~120 行）
- 新增 3 个辅助函数：`normalizeScore()`, `calculateConfidence()`, `calculateArchetypeMatch()`
- 改进模块化设计，提升代码可维护性

#### 数据利用

- 使用 23+ 个 API 字段（从原来的 12 个）
- 新增使用字段：
  - 效率指标: 净打点效率、打点效率、铳点损失
  - 条件统计: 立直后和牌率、立直后放铳率、副露后和牌率、副露后放铳率、立直收支
  - 时序指标: 和了巡数、立直巡目、副露巡目
  - 防守指标: 平均铳点、被炸率、默听率
  - 其他: 自摸率、流局率、一发率、里宝率

#### 已知限制

- 原型检测需要足够的样本量（建议 100+ 局）
- 部分玩家可能不匹配任何原型（显示原始称号）
- 危险度评分受样本量影响，小样本置信度较低

## 版本号规范

本项目采用语义化版本号（Semantic Versioning）：`vMAJOR.MINOR.PATCH`

- **MAJOR（主版本号）**：不兼容的API修改
- **MINOR（次版本号）**：向下兼容的功能性新增
- **PATCH（修订号）**：向下兼容的问题修正
