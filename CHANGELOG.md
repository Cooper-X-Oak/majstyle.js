# CHANGELOG

## [2.1.0-beta.1] - 2026-03-14 (测试版)

### Phase 1: 建议系统增强 - 测试版

⚠️ **这是测试版本，用于调试和验证功能，不是正式发布版本**

#### 新增功能

**多维度分析系统**
- ⚡ **危险度评估**: 0-10 分综合评分，基于 4 个维度（净打点效率、默听率、立直收支、平均铳点）
- 💪 **综合实力评估**: 基于净打点效率，分级：强劲/中等/较弱
- 🎭 **隐蔽性评估**: 基于默听率，识别无征兆进攻风险
- ⚡ **速度评估**: 基于和了巡数，分级：快速/中速/慢速
- 🎯 **立直质量评估**: 基于立直收支、立直后和牌率/放铳率
- 🔧 **副露质量评估**: 基于副露后和牌率/放铳率
- 🛡️ **防守能力分析**: 基于放铳率、平均铳点、被炸率

**智能策略建议**
- 动态生成 5-8 条针对性建议（替代原来的 7 条通用建议）
- 基于 7 个维度的综合分析
- 提供具体的对局策略指导

**数据利用增强**
- 使用 23 个 API 字段（从原来的 9 个）
- 新增使用 14 个高价值字段：
  - 净打点效率、打点效率、铳点损失
  - 默听率、和了巡数、立直巡目
  - 立直收支、立直后和牌率、立直后放铳率
  - 副露后和牌率、副露后放铳率
  - 平均铳点、被炸率、自摸率

#### 改进

- 增强控制台输出格式，显示更详细的分析结果
- 优化建议生成逻辑，提高针对性和实用性
- 增加 API 请求超时时间（10s → 15s）
- 添加自动重试机制（最多重试 2 次）
- 添加详细的调试日志，方便问题诊断：
  - `[开始]` 请求开始
  - `[成功]` 请求成功
  - `[调试]` 调试信息
  - `[重试]` 重试信息

#### 技术改进

- 重构 `advice-generator.js`（35 行 → ~250 行）
- 新增 8 个专业分析函数
- 改进 API 客户端的错误处理和重试逻辑
- 改进模块化设计

#### 已知问题

- API 请求可能超时（正在调试中）
- 需要验证新字段是否被 API 返回

## [2.0] - 2026-03-11

### 正式版发布

**最终可用版本**

#### 新增功能

- ✅ 实时风格分析（9种主称号 + 11种特征标签）
- ✅ 半透明悬浮窗显示（自适应宽度，60%透明度）
- ✅ 颜色编码偏差值（红=高于平均，绿=低于平均，深浅表示程度）
- ✅ 无数据玩家提示（显示"无牌谱数据"）
- ✅ 智能位置分配（自己固定左下角，其他玩家动态分配）
- ✅ 退出房间自动清除信息框
- ✅ 快速加载（2秒启动，1秒检测间隔）

#### 技术特点

- 使用 `unsafeWindow` 访问游戏对象
- 完全合规，不触碰雀魂反爬机制
- 仅调用牌谱屋公开 API
- ES5 兼容，无外部依赖

## 版本号规范

本项目采用语义化版本号（Semantic Versioning）：`vMAJOR.MINOR.PATCH`

- **MAJOR（主版本号）**：不兼容的API修改
- **MINOR（次版本号）**：向下兼容的功能性新增
- **PATCH（修订号）**：向下兼容的问题修正
