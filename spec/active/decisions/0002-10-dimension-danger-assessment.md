# ADR-0002: 采用 10 维度危险度评估系统

**状态**: 🟢 已采纳
**日期**: 2026-03-01
**决策者**: Cooper
**替代者**: N/A

## 背景
Phase 1 需要设计一个建议生成系统，帮助用户评估对手的危险程度并提供战术建议。需要在以下方面做出选择：
1. 评估维度的数量和类型
2. 评分系统的设计
3. 如何平衡全面性和可解释性

## 决策
我们决定采用 10 维度危险度评估系统，每个维度 0-10 分，通过加权计算综合危险度。

**10 个评估维度**：
1. 综合实力（基于净打点效率）
2. 隐蔽性（基于默听率）
3. 速度评级（基于和了巡数）
4. 立直质量（立直收支、立直后和牌率/放铳率）
5. 副露质量（副露后和牌率/放铳率）
6. 防守能力（放铳率、平均铳点、被炸率）
7. 进攻模式（立直率、副露率、自摸率）
8. 样本量置信度（动态权重调整）
9. 玩家原型检测（6 种核心原型）
10. 综合危险度（加权总分）

系统使用 23+ 个 API 字段进行分析。

## 理由

### 全面性
- 10 个维度覆盖了麻将对局的主要方面：进攻、防守、速度、隐蔽性
- 每个维度都有明确的统计指标支撑
- 可以识别不同类型的危险玩家（高攻击、高防守、高速度等）

### 可解释性
- 每个维度都有清晰的含义和计算方法
- 0-10 分的评分系统直观易懂
- 用户可以看到每个维度的具体得分，理解评估依据

### 扩展性
- 维度设计模块化，易于添加新维度
- 权重系统灵活，可以根据实际效果调整
- 样本量置信度系统可以动态调整权重，适应不同数据质量

## 备选方案

### 方案 A: 单一综合评分
- 优点：
  - 简单直观，一个分数表示危险度
  - 计算成本低
- 缺点：
  - 缺乏细节，无法解释为什么危险
  - 无法识别不同类型的危险玩家
  - 用户难以理解评分依据
- 为何未采纳：可解释性不足，无法提供有针对性的建议

### 方案 B: 5 维度简化系统
- 优点：
  - 比单一评分更详细
  - 计算成本适中
- 缺点：
  - 维度过少，无法全面评估
  - 可能遗漏重要特征（如隐蔽性、速度）
- 为何未采纳：全面性不足，无法满足高级用户需求

### 方案 C: 20+ 维度复杂系统
- 优点：
  - 极其全面，覆盖所有细节
  - 可以进行非常精细的分析
- 缺点：
  - 信息过载，用户难以理解
  - 计算成本高
  - 维度间可能存在冗余
- 为何未采纳：复杂度过高，用户体验差

## 后果

### 正面影响
- 提供全面的对手评估，覆盖多个关键维度
- 用户可以理解每个维度的含义和评分依据
- 系统可以识别不同类型的危险玩家，提供针对性建议
- 模块化设计便于后续扩展和优化

### 负面影响
- 计算成本比简化系统高（需要处理 23+ 个 API 字段）
- UI 需要展示 10 个维度的信息，可能占用较多空间
- 需要设计合理的权重系统，避免某些维度过度影响总分

### 权衡
我们选择了全面性和可解释性，接受了适度的计算成本和 UI 复杂度。这符合项目的目标：为高级玩家提供深度分析工具。
