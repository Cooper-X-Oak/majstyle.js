# ADR-0003: 选择 6 种核心玩家原型

**状态**: 🟢 已采纳
**日期**: 2026-03-01
**决策者**: Cooper
**替代者**: N/A

## 背景
Phase 1 建议系统需要识别玩家的打法风格，并提供针对性的战术建议。需要在以下方面做出选择：
1. 原型的数量和类型
2. 原型的识别标准
3. 如何平衡覆盖面和准确性

## 决策
我们决定采用 6 种核心玩家原型系统，每种原型有明确的识别标准和专属建议。

**6 种核心原型**：
1. **立直专家**：立直率 ≥ 30%，立直收支 > 0
2. **副露专家**：副露率 ≥ 40%，副露后和牌率 ≥ 30%
3. **默听猎手**：默听率 ≥ 15%，放铳率 < 15%
4. **速攻型**：平均和了巡数 < 11.5，和牌率 ≥ 25%
5. **价值型**：平均打点 > 7500，役满率 > 0.3%
6. **防守型**：放铳率 < 12%，被炸率 < 1.2%

每种原型都有独特的战术特征和应对策略。

## 理由

### 覆盖主流打法
- 6 种原型覆盖了四人麻将的主要打法风格
- 立直、副露、默听是三种主要进攻方式
- 速度、价值、防守是三种主要战术倾向
- 大多数玩家可以被归类到至少一种原型

### 易于识别
- 每种原型都有明确的统计指标阈值
- 识别标准基于牌谱屋 API 提供的字段
- 不需要复杂的机器学习模型
- 可以实时计算，无需额外数据

### 建议针对性强
- 每种原型都有独特的弱点和应对策略
- 可以提供具体的战术建议（如"警惕立直专家的高质量立直"）
- 用户可以根据原型调整自己的打法
- 原型信息可以与 10 维度评估结合，提供更全面的分析

## 备选方案

### 方案 A: 3 种简化原型（进攻型、防守型、平衡型）
- 优点：
  - 简单易懂
  - 覆盖所有玩家
- 缺点：
  - 过于笼统，缺乏细节
  - 无法区分不同的进攻方式（立直 vs 副露 vs 默听）
  - 建议针对性不足
- 为何未采纳：细节不足，无法提供有价值的战术建议

### 方案 B: 12+ 种详细原型
- 优点：
  - 极其详细，覆盖所有细分风格
  - 可以提供非常精准的建议
- 缺点：
  - 原型过多，用户难以记忆
  - 识别标准复杂，可能出现误判
  - 很多玩家可能不符合任何原型
- 为何未采纳：复杂度过高，实用性不足

### 方案 C: 使用机器学习聚类
- 优点：
  - 可以自动发现玩家群体
  - 不需要人工定义原型
- 缺点：
  - 需要大量训练数据
  - 模型不可解释，用户难以理解
  - 计算成本高，不适合实时分析
  - 需要定期重新训练
- 为何未采纳：技术复杂度过高，不符合项目定位

## 后果

### 正面影响
- 可以识别大多数玩家的主要打法风格
- 提供针对性的战术建议，帮助用户应对不同类型的对手
- 原型信息直观易懂，用户可以快速理解
- 识别标准明确，易于实现和维护

### 负面影响
- 部分玩家可能不符合任何原型（打法过于平衡或数据不足）
- 部分玩家可能符合多个原型（如既是立直专家又是速攻型）
- 原型识别依赖阈值设定，可能需要根据实际效果调整

### 权衡
我们选择了实用性和可解释性，接受了部分玩家无法归类的情况。系统允许一个玩家匹配多个原型，这实际上提供了更丰富的信息。
