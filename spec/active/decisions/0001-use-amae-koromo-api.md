# ADR-0001: 使用牌谱屋公开 API 而非解析游戏协议

**状态**: 🟢 已采纳
**日期**: 2021-01-01
**决策者**: Cooper
**替代者**: N/A

## 背景
雀魂（Mahjong Soul）游戏使用 WebSocket 通信和 protobuf 协议进行数据传输。为了获取玩家统计数据，有两种可能的技术路线：
1. 解析游戏的 WebSocket 通信和 protobuf 协议
2. 使用第三方公开 API（如牌谱屋 Amae Koromo）

需要在合规性、稳定性、维护成本之间做出权衡。

## 决策
我们决定使用牌谱屋（Amae Koromo）的公开 API (`https://5-data.amae-koromo.com/api/v2/pl4`) 获取玩家统计数据，而不是解析游戏协议。

脚本仅从 `window.view.DesktopMgr.Inst.player_datas`（游戏公开暴露的数据）读取玩家 ID 和昵称，然后调用外部 API 获取统计信息。

## 理由

### 合规性
- 雀魂有严格的反作弊系统，解析 WebSocket 通信或 protobuf 协议可能被视为违规行为
- 使用公开暴露的 `window` 对象和外部 API 不涉及游戏协议解析，符合游戏规则
- 牌谱屋 API 是公开可访问的，不需要认证或破解

### 稳定性
- 游戏协议可能随版本更新而变化，需要持续维护解析逻辑
- 牌谱屋 API 相对稳定，有明确的版本管理（v2）
- 外部 API 的变更不会直接影响游戏客户端

### 维护成本
- 解析 protobuf 协议需要逆向工程和持续跟踪协议变化
- 使用公开 API 只需处理 HTTP 请求和 JSON 响应
- API 文档相对清晰，易于理解和使用

## 备选方案

### 方案 A: 解析 WebSocket 通信和 protobuf 协议
- 优点：
  - 可以获取实时游戏数据（当前手牌、对手打牌等）
  - 不依赖第三方服务
  - 数据更新更及时
- 缺点：
  - 可能违反游戏规则，导致账号封禁
  - 需要逆向工程，技术难度高
  - 协议变更需要持续维护
  - 可能触发反作弊检测
- 为何未采纳：合规性风险过高，维护成本过大

### 方案 B: 使用雀魂官方 API
- 优点：
  - 官方支持，合规性有保障
  - 数据准确性高
- 缺点：
  - 雀魂未提供公开的玩家统计 API
  - 需要官方授权，不适用于第三方脚本
- 为何未采纳：官方 API 不存在或不可用

## 后果

### 正面影响
- 脚本合规，不会触发反作弊系统
- 开发和维护成本低
- 代码简洁，易于理解和扩展
- 用户使用安全，不会导致账号风险

### 负面影响
- 无法获取实时游戏数据（如当前手牌、对手打牌）
- 依赖第三方服务（牌谱屋），如果 API 不可用则功能失效
- 数据更新有延迟（牌谱屋需要时间收集和处理数据）
- 只能分析历史统计数据，无法提供实时战术建议

### 权衡
我们选择了合规性和稳定性，放弃了实时数据分析能力。这符合项目的定位：提供基于历史数据的对手风格分析，而非实时战术辅助。
