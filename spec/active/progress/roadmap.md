## Phase 3: 扩展分析维度 🟡 可选

**前置条件**: 完成 Phase 0.1，确认 API 支持更多字段

**目标**: 从 API 中提取更多有价值的指标

**状态**: 🟡 待评估

**说明**: Phase 1 已使用 23+ 个 API 字段实现 10 维度评估。本阶段将评估是否需要进一步扩展。

### 3.1 探索新指标

**已使用的指标**（Phase 1）:
- 基础统计: 对局数、和牌率、放铳率、自摸率、副露率、立直率、默听率
- 打点相关: 平均打点、平均铳点、平均自摸、平均荣和、净打点效率
- 立直相关: 立直收支、立直后和牌率、立直后放铳率
- 副露相关: 副露后和牌率、副露后放铳率
- 防守相关: 被炸率、追立率
- 速度相关: 平均和了巡数

**可能的新指标**（需验证 API 支持）:
- 一发率、里宝率
- 流局率、听牌率
- 平均顺位、1位率、2位率、3位率、4位率
- 连庄率、被飞率

### 3.2 实现新分析维度

**文件**: `src/analysis/style-analyzer.js`, `src/analysis/advice-generator.js`

**任务**:
- 添加新指标的分析函数
- 更新风格分类逻辑
- 更新建议生成逻辑
- 创建 ADR 记录新维度的决策理由

**预计工作量**: 4-6 小时（取决于新指标数量）

### 3.3 趋势分析

**前置条件**: API 支持时间范围查询

**任务**:
- 实现最近 100 局 vs 全部数据对比
- 显示数据变化趋势
- 在建议中体现趋势信息
- 更新设计文档

**预计工作量**: 3-4 小时

**总预计工作量**: 7-10 小时（可选）

## 风险和挑战

### 技术风险

1. **API 数据限制** ✅ 已缓解
   - 风险: API 可能不提供足够的字段
   - 缓解: Phase 0 已完成探索，确认 51 个字段可用
   - 状态: Phase 1 成功使用 23+ 个字段实现 10 维度评估

2. **localStorage 容量限制** 🟡 待评估
   - 风险: 档案数据可能超过 5MB 限制
   - 缓解: 限制快照数量，定期清理旧数据
   - 状态: Phase 2 设计阶段需详细评估

3. **游戏对象不稳定** ✅ 已缓解
   - 风险: 游戏更新可能导致对象结构变化
   - 缓解: 添加错误处理，优雅降级
   - 状态: 已在 `src/game/game-bridge.js` 中实现错误处理

4. **实时数据收集的合规性** 🟡 待决策
   - 风险: 实时数据收集可能触发反作弊检测
   - 缓解: 提案阶段充分评估，必要时放弃该功能
   - 状态: 提案 `spec/proposals/20260315-realtime-data-collection.md` 待评审
   - 相关决策: `spec/active/decisions/0001-use-amae-koromo-api.md`

### 用户体验风险

1. **建议质量不高** ✅ 已缓解
   - 风险: 建议不够准确或实用
   - 缓解: 收集用户反馈，持续优化
   - 状态: Phase 1 已实现 10 维度评估和 6 种原型检测，建议质量显著提升

2. **性能影响** 🟡 待监控
   - 风险: 档案系统可能影响性能
   - 缓解: 异步处理，延迟加载
   - 状态: Phase 2 实施时需重点关注

3. **UI 信息过载** ✅ 已缓解
   - 风险: 10 维度信息可能导致 UI 过于复杂
   - 缓解: 仅显示最重要的 3 条建议，详细信息在控制台
   - 状态: Phase 1 已实现简洁的 UI 设计
