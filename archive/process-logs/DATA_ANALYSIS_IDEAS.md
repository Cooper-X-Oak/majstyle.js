## 分析维度设计

### 维度 1: 进攻意愿分析

**使用字段**:
- 立直率
- 副露率

**计算方法**:
```javascript
进攻意愿 = (立直率 + 副露率) / 2
```

**分类标准**:
- 高意愿: > 基准 + 5%
- 正常: 基准 ± 5%
- 低意愿: < 基准 - 5%

**实战价值**:
- 判断对手的进攻倾向
- 决定是否需要加快节奏

### 维度 2: 进攻效率分析

**使用字段**:
- 和牌率
- 立直率
- 副露率

**计算方法**:
```javascript
进攻效率 = 和牌率 / (立直率 + 副露率)
```

**分类标准**:
- 高效率: > 基准 + 10%
- 正常: 基准 ± 10%
- 低效率: < 基准 - 10%

**实战价值**:
- 评估对手的进攻质量
- 判断是否可以利用其失误

### 维度 3: 防守能力分析

**使用字段**:
- 放铳率

**分类标准**:
- 铁壁型: < 基准 - 2%
- 正常: 基准 ± 2%
- 漏勺型: > 基准 + 2%

**实战价值**:
- 判断施压的成功率
- 决定进攻的激进程度

### 维度 4: 立直质量分析

**使用字段**:
- 追立率
- 先制率
- 立直好型率

**分类标准**:
- 追立狂魔: 追立率 > 25%
- 先制王: 先制率 > 85%
- 好型立直: 立直好型 > 60%
- 赌博立直: 立直好型 < 40%

**实战价值**:
- 判断对手立直的危险程度
- 决定是否弃牌或对抗

### 维度 5: 打点能力分析

**使用字段**:
- 平均打点

**分类标准**:
- 大牌猎人: > 基准 + 500
- 正常: 基准 ± 500
- 速和型: < 基准 - 500

**实战价值**:
- 评估放铳的损失风险
- 决定防守的优先级

## 风险评估系统

### 危险度评分公式

```javascript
危险度 = (
  进攻意愿权重 * 进攻意愿分数 +
  进攻效率权重 * 进攻效率分数 +
  防守能力权重 * (10 - 防守能力分数) +
  打点能力权重 * 打点能力分数
) / 总权重
```

**权重设定**（待实际测试调整）:
- 进攻意愿: 2
- 进攻效率: 3
- 防守能力: 2
- 打点能力: 3

**评分标准**:
- 0-2: 🎯 送分目标
- 3-4: ✓ 威胁较小
- 5-6: ➡️ 正常水平
- 7-8: ⚡ 较为危险
- 9-10: ⚠️ 极度危险

## 建议生成策略

### 策略 1: 进攻建议

**触发条件**: 根据进攻意愿 × 进攻效率矩阵

**高意愿高效率**:
- "对手进攻意愿强且效率高，正面对抗风险大"
- "建议抢先进攻，不给对手机会"
- "如果对手先制立直，建议立即弃牌"

**高意愿低效率**:
- "对手频繁进攻但效率不高，容易失误"
- "可以适度施压，利用其激进打法"
- "注意防守，但不必过度保守"

**低意愿高效率**:
- "对手不常进攻，但出手必中，极度危险"
- "一旦对手有进攻迹象，立即提高警惕"
- "建议优先防守，避免放铳"

**低意愿低效率**:
- "对手威胁较小，可以正常打法"
- "适度施压，争取主动权"

### 策略 2: 防守建议

**触发条件**: 根据防守类型

**铁壁型（放铳率低）**:
- "对手防守能力强，难以攻破"
- "需要更高的牌型质量才能和牌"
- "不建议过度激进，避免无效进攻"

**漏勺型（放铳率高）**:
- "对手容易放铳，是送分目标"
- "积极施压，增加对手失误概率"
- "可以适度降低和牌标准，追求速度"

### 策略 3: 立直应对建议

**触发条件**: 根据立直特征

**追立狂魔**:
- "对手喜欢追立，敢于对攻"
- "你立直后要警惕对手追立"
- "对手立直时可能是追立，危险度中等"

**先制王**:
- "对手很少追立，立直多为先制"
- "对手立直时大概率是先制，危险度高"
- "建议立即弃牌，避免放铳"

**好型立直**:
- "对手立直质量高，听牌好型多"
- "对手立直后建议立即弃牌"
- "不建议对抗，放铳风险大"

**赌博立直**:
- "对手立直质量不佳，可能是赌博立直"
- "可以适度对抗，不必立即弃牌"
- "注意观察牌河，判断听牌范围"

### 策略 4: 打点策略建议

**触发条件**: 根据平均打点

**大牌猎人**:
- "对手平均打点高，警惕满贯以上"
- "放铳损失大，优先防守"
- "如果对手立直，建议立即弃牌"

**速和型**:
- "对手追求速度，打点较低"
- "放铳损失相对较小，可以适度对抗"
- "注意不要被对手频繁和牌拉开差距"

## 综合建议生成逻辑

### 优先级排序

1. **最高优先级**: 危险度评估
2. **高优先级**: 立直应对建议（如果对手立直率高）
3. **中优先级**: 进攻策略建议
4. **中优先级**: 防守策略建议
5. **低优先级**: 打点策略建议

### 建议数量控制

- UI 卡片显示: 最重要的 3 条
- 控制台输出: 完整的所有建议（分类显示）

### 建议去重

避免重复或矛盾的建议：
- 如果已经提示"极度危险"，不再重复"建议立即弃牌"
- 如果已经提示"送分目标"，不再重复"威胁较小"

---

## 数据不足处理

### 对局数过少

**判断标准**: 对局数 < 50

**处理方式**:
- 显示警告: "⚠️ 数据量不足（仅 X 局），分析可能不准确"
- 降低建议的确定性: "可能..." 而不是 "建议..."
- 不显示危险度评分

### 数据缺失

**判断标准**: 某些字段为 null 或 undefined

**处理方式**:
- 跳过该维度的分析
- 在建议中说明: "（部分数据缺失，分析不完整）"
