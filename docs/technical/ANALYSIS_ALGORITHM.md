## 分析维度

### 1. 风格分析（Style Analysis）

**目标**: 根据玩家的进攻意愿、进攻效率、防守能力、打点倾向，生成主称号和标签。

#### 1.1 进攻意愿

**计算公式**:
```
进攻意愿 = 立直率 + 副露率
基准进攻意愿 = 段位基准立直率 + 段位基准副露率
进攻意愿偏差 = 进攻意愿 - 基准进攻意愿
```

**分类阈值**（可配置）:
- 高: 偏差 > 1.5
- 中: -1.5 ≤ 偏差 ≤ 1.5
- 低: 偏差 < -1.5

**推理逻辑**: 立直和副露都是主动进攻的表现，两者之和反映玩家的进攻意愿。

#### 1.2 进攻效率

**计算公式**:
```
进攻效率 = (和牌率 / 进攻意愿) × 100
基准进攻效率 = (段位基准和牌率 / 基准进攻意愿) × 100
进攻效率偏差 = 进攻效率 - 基准进攻效率
```

**分类阈值**（可配置）:
- 高: 偏差 > 2.0
- 中: -2.0 ≤ 偏差 ≤ 2.0
- 低: 偏差 < -2.0

**推理逻辑**: 进攻效率反映玩家进攻的成功率，高效率说明玩家能有效转化进攻意愿为和牌。

#### 1.3 防守能力

**计算公式**:
```
放铳率偏差 = 放铳率 - 段位基准放铳率
```

**分类阈值**（可配置）:
- 铁壁: 偏差 < -2.0
- 正常: -2.0 ≤ 偏差 ≤ 2.0
- 漏勺: 偏差 > 2.0

**推理逻辑**: 放铳率直接反映防守能力，低放铳率说明防守强。

#### 1.4 打点倾向

**计算公式**:
```
打点偏差 = 平均打点 - 段位基准平均打点
```

**分类阈值**（可配置）:
- 大牌: 偏差 > 500
- 均衡: -500 ≤ 偏差 ≤ 500
- 速和: 偏差 < -500

**推理逻辑**: 平均打点反映玩家追求大牌还是速和，高打点说明倾向做大牌。

#### 1.5 主称号生成

**称号映射表**（可配置）:

| 进攻意愿 | 防守能力 | 主称号 |
|----------|----------|--------|
| 高 | 铁壁 | 钢铁战士 |

| 高 | 漏勺 | 自爆兵 |

| 中 | 正常 | 上班族 |

| 低 | 正常 | 摆烂人 |

**标签触发条件**（可配置）:

### 2. 综合实力评估

**目标**: 基于净打点效率评估玩家的综合实力。

**计算公式**:
```
净打点效率 = 打点效率 - 铳点损失
```

**分类阈值**（可配置）:
- 强劲: 净打点效率 > 500
- 中等: 0 ≤ 净打点效率 ≤ 500
- 较弱: 净打点效率 < 0

**推理逻辑**: 净打点效率综合反映玩家的进攻和防守能力，正值说明得点大于失点，实力强劲。

---

### 3. 隐蔽性评估

**目标**: 基于默听率评估玩家的隐蔽性。

**计算公式**:
```
默听率 = 和牌率 - 立直率 - 副露率
```

**分类阈值**（可配置）:
- 高隐蔽: 默听率 > 15%
- 中等: 8% ≤ 默听率 ≤ 15%
- 低隐蔽: 默听率 < 8%

**推理逻辑**: 默听率反映玩家不鸣牌、不立直而直接和牌的频率，高默听率说明进攻意图隐蔽。

---

### 4. 速度评估

**目标**: 基于和了巡数评估玩家的速度。

**分类阈值**（可配置）:
- 快速: 和了巡数 < 11
- 中速: 11 ≤ 和了巡数 ≤ 13
- 慢速: 和了巡数 > 13

**推理逻辑**: 和了巡数反映玩家完成和牌的速度，低巡数说明速度快。

---

### 5. 立直质量评估

**目标**: 基于立直收支、立直后和牌率、立直后放铳率评估立直质量。

**分类阈值**（可配置）:
- 高质量: 立直收支 > 2000 且 立直后和牌率 > 45%
- 中等: 立直收支 > 0
- 低质量: 立直收支 < 0

**推理逻辑**: 立直收支 = 立直后和牌得点 - 立直后放铳失点，高收支说明立直质量优秀。

---

### 6. 副露质量评估

**目标**: 基于副露后和牌率、副露后放铳率评估副露质量。

**分类阈值**（可配置）:
- 高效: 副露后和牌率 > 35% 且 副露后放铳率 < 15%
- 中等: 副露后和牌率 > 28%
- 低效: 副露后和牌率 < 28%

**推理逻辑**: 副露后和牌率反映副露的成功率，高和牌率说明副露判断准确。

---

### 7. 进攻特征分析

**目标**: 识别玩家的进攻模式。

**分类逻辑**（可配置）:
- 立直主导: 立直率 > 25% 且 副露率 < 30%
- 副露主导: 副露率 > 38% 且 立直率 < 20%
- 混合进攻: 立直率 > 20% 且 副露率 > 30%
- 保守进攻: 其他情况

---

### 8. 防守能力分析

**目标**: 评估玩家的防守等级。

**分类阈值**（可配置）:
- 强: 放铳率 < 13% 且 平均铳点 < 5000
- 中等: 放铳率 < 15%
- 弱: 放铳率 ≥ 15%

**大牌防守评估**:
- 好: 被炸率 ≤ 10%
- 差: 被炸率 > 10%

## 原型检测

**目标**: 识别玩家是否符合 6 种核心原型之一。

### 原型定义

#### 1. 立直专家（🎯）

**匹配条件**:
- 立直率 ≥ 25%
- 立直收支 ≥ 1500
- 副露率 ≤ 30%

**战术建议**:
- 对手立直质量高，立直后建议立即弃牌
- 警惕对手的好型立直，避免放铳大牌
- 对手不擅长副露，可以通过鸣牌抢先

#### 2. 副露专家（⚡）

**匹配条件**:
- 副露率 ≥ 38%
- 副露后和牌率 ≥ 30%
- 立直率 ≤ 20%

**战术建议**:
- 对手副露效率高，副露后威胁大
- 注意对手的速攻倾向，需要抢先进攻
- 对手立直较少，可以通过立直施压

#### 3. 默听猎手（🥷）

**匹配条件**:
- 默听率 ≥ 15%
- 立直率 ≤ 20%
- 放铳率 ≤ 14%

**战术建议**:
- 对手有高默听倾向，警惕无征兆进攻
- 对手防守能力强，需要更高牌型质量
- 难以通过立直/副露判断对手听牌状态

#### 4. 速攻型（💨）

**匹配条件**:
- 和了巡数 ≤ 11
- 副露率 ≥ 35%
- 和牌率 ≥ 21%

**战术建议**:
- 对手速度极快，需要抢先进攻
- 对手倾向速和小牌，可以做大牌反制
- 警惕对手的快速副露进攻

#### 5. 价值型（💎）

**匹配条件**:
- 平均打点 ≥ 7000
- 立直率 ≥ 20%
- 和了巡数 ≥ 12

**战术建议**:
- 对手倾向做大牌，防守时要特别小心
- 对手速度较慢，有时间布局和抢先
- 对手立直质量高，立直后建议弃牌

#### 6. 防守型（🛡️）

**匹配条件**:
- 放铳率 ≤ 13%
- 被炸率 ≤ 8%
- 默听率 ≥ 12%

**战术建议**:
- 对手防守能力极强，难以放铳
- 对手倾向默听，进攻意图不明显
- 需要更高的牌型质量才能有效施压

### 匹配算法

**步骤**:
1. 遍历所有原型定义
2. 检查玩家数据是否满足所有条件
3. 计算匹配度得分（基于偏差程度）
4. 返回匹配度最高的原型

**匹配度计算**:
```javascript
对于每个条件:
  如果满足条件:
    得分 += 1 + min(偏差程度, 0.5)
  否则:
    得分 -= 偏差程度

如果所有条件都满足:
  返回得分
否则:
  返回 0（不匹配）
```

## 危险度评分

**目标**: 综合评估对手的威胁程度（0-10 分）。

### 评分维度（10 个）

| 维度 | 权重 | 计算方式 | 说明 |
|------|------|----------|------|
| 净打点效率 | 30% | 归一化(-1000, 2000) → (0, 10) | 综合实力指标 |
| 默听率 | 15% | 默听率 × 50 | 隐蔽性/危险性 |
| 立直收支 | 20% | 归一化(-1000, 3000) → (0, 10) | 立直质量 |
| 平均铳点 | 5% | 10 - 归一化(4000, 6000) → (0, 10) | 防守弱点（反向） |
| 被炸率 | 10% | 10 - (被炸率 × 100) | 大牌防守（反向） |
| 和了巡数 | 10% | 10 - 归一化(9, 14) → (0, 10) | 速度（反向） |
| 立直后和牌率 | 5% | 立直后和牌率 × 20 | 立直效率 |
| 副露后和牌率 | 3% | 副露后和牌率 × 25 | 副露效率 |
| 和牌率 | 2% | 归一化(18, 26) → (0, 10) | 基础进攻能力 |

### 置信度调整

**样本量置信度**:
- ≥ 400 局: 100%
- ≥ 200 局: 95%
- ≥ 100 局: 90%
- ≥ 50 局: 80%
- < 50 局: 70%

**最终危险度**:
```
危险度 = (加权得分 / 总权重) × 置信度
危险度 = round(min(10, max(0, 危险度)))
```

### 危险度标签

| 分数 | 图标 | 标签 |
|------|------|------|
| 9-10 | ⚠️ | 极度危险 |
| 7-8 | ⚡ | 较为危险 |
| 5-6 | ➡️ | 正常水平 |
| 3-4 | ✓ | 威胁较小 |
| 0-2 | 🎯 | 送分目标 |

## 建议生成

**目标**: 基于分析结果生成结构化的战术建议。

### 建议来源（8 个维度）

1. **玩家原型**: 如果匹配原型，优先添加原型特定建议
2. **综合实力**: 基于净打点效率
3. **隐蔽性**: 基于默听率
4. **速度**: 基于和了巡数
5. **立直质量**: 基于立直收支
6. **副露质量**: 基于副露后和牌率
7. **防守能力**: 基于放铳率和平均铳点
8. **大牌防守**: 基于被炸率

### 建议结构

每条建议包含：
- **建议**: 简短的战术建议（1 句话）
- **理由**: 详细的推理过程（包含触发条件、数据支撑、推理逻辑、战术含义）
- **置信度**: 基于样本量（高/中/低）
- **优先级**: 0-10 分（基于偏差程度）
- **来源**: 建议来源维度

### 建议优先级计算

```javascript
优先级 = round((偏差 - 最小阈值) / (最大阈值 - 最小阈值) × 10)
优先级 = min(10, max(0, 优先级))
```

### 建议去重和排序

1. 按优先级降序排序
2. 去除重复建议（相同建议文本）
3. 最多返回 8 条建议

### 如何修改配置

#### 方法 1: 直接修改配置文件

1. 编辑 `src/config/analysis-config.json`
2. 运行 `npm run build` 重新构建
3. 刷新游戏页面

#### 方法 2: 使用 localStorage 覆盖（热更新）

在浏览器控制台执行：

```javascript
// 保存自定义配置
var customConfig = {
  "style_thresholds": {
    "进攻意愿": { "高": 2.0, "低": -2.0 }
  }
};
localStorage.setItem('majstyle_custom_config', JSON.stringify(customConfig));

// 清除自定义配置
localStorage.removeItem('majstyle_custom_config');
```

1. 在 `dimensions` 中添加新维度定义
2. 在分析模块中实现新维度的计算逻辑
3. 在 UI 中显示新维度的结果

### 归一化函数

```javascript
function normalizeScore(value, min, max, outMin, outMax) {
  var normalized = (value - min) / (max - min);
  normalized = Math.min(1, Math.max(0, normalized));
  return outMin + normalized * (outMax - outMin);
}
```
