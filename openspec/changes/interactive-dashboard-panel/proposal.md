## Why

当前的雀魂数据分析助手只能在对局开始时自动显示对手风格信息，缺乏主动查询和深度对比能力。玩家需要一个交互式工具来：(1) 随时搜索任意玩家的历史数据，(2) 通过可视化图表对比多个玩家的风格差异，(3) 在不离开游戏的情况下进行数据分析。这将显著提升玩家的策略决策能力和游戏体验。

## What Changes

- 新增可拖拽的悬浮面板，支持快捷键（Ctrl+Shift+D）和悬浮按钮触发
- 新增玩家搜索功能，支持 ID/昵称搜索和一键添加当前对局玩家
- 新增雷达图可视化，支持 2-4 个玩家的 6 维度数据叠加对比
- 新增柱状图可视化，横向对比多个玩家的关键指标
- 新增多玩家对比列表，最多同时对比 4 个玩家
- 集成 Chart.js 图表库（CDN 引入）
- 新增数据缓存机制（5 分钟有效期）和用户偏好存储（localStorage）
- 扩展现有的牌谱屋 API 调用逻辑，支持批量请求优化

## Capabilities

### New Capabilities

- `draggable-panel`: 可拖拽、可调整大小的悬浮面板，支持最小化/最大化，记住用户偏好
- `player-search`: 玩家搜索功能，支持 ID/昵称搜索、模糊匹配、快速添加当前对局玩家
- `radar-chart-visualization`: 6 维度雷达图可视化，支持 2-4 个玩家叠加显示，显示段位基准线
- `bar-chart-visualization`: 柱状图可视化，横向对比多个玩家的关键指标，颜色编码偏差值
- `multi-player-comparison`: 多玩家对比功能，管理对比列表（最多 4 个），高亮差异指标

### Modified Capabilities

- `player-style-analysis`: 从自动显示扩展为支持主动查询和批量对比，增加数据缓存机制

## Impact

**代码影响**
- 扩展现有的 `雀魂金玉四麻风格分析助手.user.js`（468 行）
- 新增约 800-1000 行代码（面板 UI、图表模块、搜索模块、对比模块）
- 重构数据获取逻辑，从单次请求改为支持批量请求和缓存

**依赖影响**
- 新增外部依赖：Chart.js v3.9.1（CDN 引入）
- 新增浏览器 API 依赖：ResizeObserver（响应式图表）

**性能影响**
- 内存占用增加：从 <5MB 增加到 <20MB（图表渲染和数据缓存）
- 启动时间：面板打开 <200ms，搜索响应 <500ms，图表渲染 <300ms
- API 请求优化：通过缓存减少重复请求，批量请求提升效率

**用户体验影响**
- 从被动接收信息转变为主动探索数据
- 增加学习曲线（需要理解面板操作和图表解读）
- 提供更深度的数据分析能力
